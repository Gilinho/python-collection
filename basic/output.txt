          <h1 class="title node-teaser-page" id="page-title">
          <a href="/install-mongodb-ubuntu-1204-precise-pangolin">Install MongoDB in Ubuntu 12.04 Precise Pangolin</a>
        </h1>

   
      <div class="meta submitted">
            <span property="dc:date dc:created" content="2012-05-12T00:41:30+07:00" datatype="xsd:dateTime" rel="sioc:has_creator">Submitted by <span class="username" xml:lang="" about="/users/admin" typeof="sioc:UserAccount" property="foaf:name">admin</span> on Sat, 05/12/2012 - 00:41</span>    </div>
  
  <div class="content clearfix">
    <div class="field field-name-body field-type-text-with-summary field-label-hidden"><div class="field-items"><div class="field-item even" property="content:encoded"><p>Here are shortcuts to install and run MongoDB in your Ubuntu 12.04 Precise Pangolin.</p>
<p>1. Add GPG Key</p>
<div class="codeblock"><code>sudo apt-key adv --keyserver keyserver.ubuntu.com --recv 7F0CEB10</code></div>
<p>2. Edit your source.list<br />
sudo vim /etc/apt/sources.list and add this into bottom line :</p>
<div class="codeblock"><code>deb http://downloads-distro.mongodb.org/repo/ubuntu-upstart dist 10gen</code></div>

      <h1 class="entry-title"><a href="/blog/2011/10/26/build-base-box-vagrant-ubuntu-oneiric-11.10-server/">Build Base Box Vagrant Ubuntu Oneiric 11.10 Server</a></h1>
    
    
      <p class="meta">
        




  

<time datetime="2011-10-26T23:26:00+08:00" pubdate>Oct 26<span>th</span>, 2011</time>
        
      </p>
    
  </header>


  <div class="entry-content"><p>Virtual Machine (VM) is good solution to handle several different environment of applications.
All my server working on Ubuntu Server with different version.
At this point, virtualenv and RVM still doesn&#8217;t enough for me.</p>

<p>So I use Virtualbox and install several Ubuntu Server version.
Then I hear about Vagrant which a great tools for managing Virtualbox VM.
Using Vagrant, we can run VM without execute Virtualbox because it running on background.
This is what i want!</p>

<p>I will explain my steps bundling Ubuntu 11.10 into Vagrant box and run it.</p>

<h3>1. Create Ubuntu 11.10 VM on Virtualbox</h3>

<p>Create new VM with name &#8220;vagrant-oneiric&#8221; and start install Ubuntu Server 11.10.
In this steps, I disable USB and Audio which not used in Server.
Also, I set Video size to 4MB only which server doesn&#8217;t run GUI.</p>

<h3>2. Installing Ubuntu 11.10</h3>

<p>Because we will use Vagrant, then we should follow standard convention here.
Make sure you setup this :</p>

<pre><code>hostname : vagrant
username : vagrant
password : vagrant
</code></pre>

<p>Also, you must  install OpenSSH-Server packages.</p>

<time datetime="2011-09-20T14:28:00+08:00" pubdate>Sep 20<span>th</span>, 2011</time>

  <div class="entry-content"><p>Running NowJS on port 80 in sub-domain is the best way to make our chat application works in every environment.
We know some &#8220;draconian office&#8221; only open port 80 &amp; 443 for internet access.
By using HAProxy, we can redirect NowJs apps on port 8003 to 80 on Sub-domain.</p>

<p>In this example, i have NGINX run on port 81 and nowJS on port 8003.
My chat homepage located in <a href="http://www.obrool.com">http://www.obrool.com</a> which rely on NGINX port 81.
My nowjs apps (8003) will pointed into <a href="http://chat.obrool.com">http://chat.obrool.com</a> on port 80.</p>

      <h1 class="entry-title"><a href="/blog/2011/08/21/how-to-solve-error-cannot-find-module-less-css-in-node-js-0-5-5/">How to Solve Error Cannot Find Module Less CSS in Node.js 0.5.5</a></h1>
    
    
      <p class="meta">
        




  

<time datetime="2011-08-21T22:00:00+08:00" pubdate>Aug 21<span>st</span>, 2011</time>
        
      </p>
    
  </header>


  <div class="entry-content"><p>Installing Less.js (fresh from <a href="https://github.com/cloudhead/less.js">cloudhead&#8217;s github</a>)
probably making some error with your nodeJS application if you install it <strong>not in global mode</strong>.</p>

<p>Here are several thing that could be happen :</p>

<h3>1. Error: require.paths is removed. Use node_modules folders, or the NODE_PATH environment</h3>

<pre><code>Error: require.paths is removed. Use node_modules folders, or the NODE_PATH environment variable instead.    
at Function.&lt;anonymous&gt; (module.js:360:11)  
at Object.&lt;anonymous&gt; (/home/ubuntu/www/js/nodejs/node_modules/less/lib/less/index.js:5:8)
at Module._compile (module.js:416:26)  
at Object..js (module.js:434:10)   
at Module.load (module.js:335:31) 
at Function._load (module.js:294:12) 
at Module.require (module.js:341:17)  
at require (module.js:352:17)  
at Object.&lt;anonymous&gt; (/home/ubuntu/www/js/nodejs/myapp.js:29:12)
at Module._compile (module.js:416:26)
</code></pre>

<h3>2. Can&#8217;t find module &#8216;less/parser&#8217;</h3>

<pre><code>Error: Cannot find module 'less/parser'  
at Function._resolveFilename (module.js:318:11)   
at Function._load (module.js:263:25)  
at Module.require (module.js:341:17)   
at require (module.js:352:17)   
at Object.&lt;anonymous&gt; (/home/ubuntu/local/node/lib/node_modules/less/lib/less/index.js:9:13) 
at Module._compile (module.js:416:26) 
at Object..js (module.js:434:10) 
at Module.load (module.js:335:31)
at Function._load (module.js:294:12)  
at Module.require (module.js:341:17)
</code></pre>

<p>So how to make this LessJS working with NodeJS ?</p>

<p>Actually problem is comming from inside Less.js code.</p>

<pre><code>    require.paths.unshift(path.join(__dirname, '..'));
</code></pre>

<p>So, let fix it!Let make some parameters here.</p>

<ol>
<li><p>I using npm (without global) when installing less.js.
I put it on <code>/home/ubuntu/www/nodejs/</code></p></li>
<li><p>I installing nodejs without sudo and install it on ~/local.
In case you don&#8217;t get it, check out my <a href="http://yoodey.com/how-install-nodejs-ubuntu-natty-1104-and-start-learning">Node.JS Installation</a>.</p></li>
</ol>


<p>So, first thing to do is editing 2 less files :</p>

<ol>
<li><p>index.js in node_modules folder
vim less.js/lib/less/index.js and comment line number 5 :</p>

<pre><code>  //require.paths.unshift(path.join(__dirname, '..'));
</code></pre></li>
<li><p>Do as number 1 on &#8220;less.js/bin/lessc&#8221; by commenting line number 7 :</p>

<pre><code> //require.paths.unshift(path.join(__dirname, '..', 'lib'));
</code></pre></li>
<li><p>Build ~/.profile to pointing into your LessJS module</p>

<pre><code> Create this on your ~/.profile:
 export PATH=$HOME/local/node/bin:$PATH
 export NODE_PATH=/home/ubuntu/node/lib/node_modules:$NODE_PATH
 export NODE_PATH=/home/ubuntu/www/nodejs/obrool/node_modules:$NODE_PATH
 export NODE_PATH=/home/ubuntu/www/nodejs/obrool/node_modules/less.js/lib:$NODE_PATH
</code></pre></li>
</ol>


<p>In this example, my less.js located in &#8220;/home/ubuntu/www/nodejs/obrool/node_modules&#8221;.
You can change it depend in your installation method and folder structure.</p>

<p>Last step &#8220;source ~/.profile&#8221; to load it NODE_PATH.</p>

<p>Start your node server!</p>

<p>Solution was tested and taken from <a href="https://github.com/cloudhead/less.js/issues/320">here</a>.</p>

<p>If youre install Less in Global mode, then you should do point number 3 which pointing into you global node_modules folder.</p>

<p>Thanks folks!</p>
</div>
